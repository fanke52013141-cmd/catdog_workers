# 一致性校验清单

## 角色一致性

- [ ] 呼噜噜是否保留猫系低电量特征？
- [ ] 呼噜噜发卡状态是否与剧情一致？
- [ ] 呼噜噜台词是否软糯、短句、迷糊，不鸡血过度？
- [ ] 啊呜呜是否保留狗系吃货碎嘴特征？
- [ ] 啊呜呜尾巴状态是否与剧情一致？
- [ ] 啊呜呜台词是否嘴快、碎碎念、有行动力？
- [ ] 两人关系是否体现互相嫌弃但互相关心？

## 内容一致性

- [ ] 视频是否只讲一个核心冲突？
- [ ] 前 3 秒是否有明确钩子？
- [ ] 结尾是否有反转、固定梗或轻治愈？
- [ ] 是否有可评论、可转发的共鸣点？
- [ ] 是否避免“AI 炫技大于内容”？

## 资产复用

- [ ] 是否调用已有角色资产？
- [ ] 是否调用已有场景资产？
- [ ] 是否产生了新的可复用资产？
- [ ] 缺失资产是否记录到 `docs/missing-assets-and-decisions.md`？

## 商业与平台风险

- [ ] 是否避免攻击具体公司或具体人？
- [ ] 是否避免过强负面情绪？
- [ ] 是否避免低俗、擦边或不适合 IP 长期化的表达？
- [ ] 如有品牌/产品出现，是否符合角色和账号调性？
