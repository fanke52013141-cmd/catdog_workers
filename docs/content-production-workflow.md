# 内容创作流程与短视频生存法则

本文件固化的是创作流程和短视频结构，不固化具体表现形式。呼噜噜可以吐槽、变身、做梦、穿越、看热点、和啊呜呜对话，但每条视频都必须服务完播、共鸣、记忆点和可持续生产。每条视频控制在 1 分钟以内，普通内容建议 15-45 秒。

## 1. 核心创作原则

一句话：

> 用短视频的生存结构，装进打工人的真实情绪，再用呼噜噜的 IP 视角和 AI 视觉把它放大。

每条内容都要回答 4 个问题：

1. 用户为什么前 3 秒不划走？
2. 用户为什么中间愿意继续看？
3. 用户为什么结尾有情绪反应？
4. 用户看完后记住了呼噜噜的什么？

## 2. 单条视频标准结构

### 0-3 秒：钩子

目标：让用户瞬间知道“这和我有关”。

钩子来源：

- 热点：最近大家都在刷什么。
- 共鸣：打工人都经历过什么。
- 反常：呼噜噜今天变成了什么。
- 悬念：她为什么电量突然归零。
- 视觉奇观：AI 变身、巨大特效、梦境画面。

钩子模板：

- “打工人看到这个热点的真实反应……”
- “如果周一早上有实体，应该长这样。”
- “呼噜噜今天醒来，发现自己只剩 3% 电量。”
- “下班前 5 分钟，最可怕的事发生了。”
- “这个梗放到工位上，突然就不搞笑了。”

### 4-12 秒：进入情绪

目标：把热点或场景翻译成打工人的具体感受。

这一段不能只解释设定，要让用户进入情绪：

- 累。
- 没电。
- 想下班。
- 想睡觉。
- 想变有钱。
- 想认真生活但今天先活着。

### 13-40 秒：升级或变化

目标：防止中段掉完播。

可以用：

- 状态升级：发卡从绿到红再到灰。
- 画面升级：现实变魔幻，工位变宇宙，咖啡变能量站。
- 情绪升级：从忍一忍到彻底宕机。
- 关系升级：啊呜呜出现，打断、吐槽、投喂或拉走。
- 反差升级：嘴上说努力，身体已经趴下。

### 41-60 秒：回收与记忆点

目标：用一句话、一个动作或一个状态让用户带着情绪离开。

可回收的 IP 记忆点：

- “电量过低，但还没关机。”
- 发卡归零。
- 抱咖啡续命。
- “躺平专员”工牌特写。
- 黑眼圈变重。
- 啊呜呜递狗骨头饼干/奶茶。
- 两人一起说“走，去充电。”

结尾不是必须正能量，但要有一点“还活着”的余温。

## 3. 内容生产全流程

### Step 1：选题池

来源：

- 热点：抖音热榜、微博热搜、小红书趋势、B站热梗。
- 日常：上班、下班、通勤、午休、工资、加班、周一周五。
- 魔幻：变身、梦境、穿越、巨大化、缩小、困在表格里。
- 关系：呼噜噜与啊呜呜的朋友分享、互相吐槽、互相投喂。

输出：一句话选题。

例子：

> 最近大家都在说“班味”，呼噜噜发现自己的班味已经实体化了。

### Step 2：选题打分

每个选题按 5 项打分：

| 维度 | 分值 | 判断 |
| --- | --- | --- |
| 热度/新鲜度 | 1-5 | 现在发有没有时效 |
| 共鸣强度 | 1-5 | 打工人会不会说“这是我” |
| IP 适配度 | 1-5 | 能不能自然落到呼噜噜/啊呜呜 |
| 视觉表现力 | 1-5 | 是否适合 AI 画面发挥 |
| 制作可控度 | 1-5 | 24-48 小时内能否完成 |

21 分以上优先做。低于 16 分先放入备选。

### Step 3：确定一句话表达

每条视频先写一句“这条到底替打工人说什么”。

格式：

```text
这条视频想表达：打工人面对【事件/热点】时，其实是在感受【情绪】，呼噜噜用【IP 反应】替大家说出来。
```

例子：

```text
这条视频想表达：打工人面对周一早晨时，其实是在感受身体和灵魂不同步，呼噜噜用“只剩 3% 电量”的变身替大家说出来。
```

### Step 4：选择表现形式

形式不固定，根据选题选择：

- 低电量吐槽。
- 魔幻变身。
- 梦境穿越。
- 工位小剧场。
- 双人对话。
- 热点魔改。
- 表情包短梗。
- 伪纪录观察。

选择标准：哪种形式最能提高前 3 秒吸引力和中段完播，就用哪种。

### Step 5：写 1 分钟以内脚本

必须包含：

- 3 秒钩子。
- 一个核心情绪。
- 一个中段变化。
- 一个 IP 记忆点结尾。

禁止：

- 一条视频讲多个观点。
- 解释太多设定。
- 纯口播没有画面变化。
- 结尾只剩鸡汤。

### Step 6：拆分镜与提示词

每 3-6 秒一个镜头。

每个镜头必须写清：

- 角色状态。
- 场景。
- 动作。
- 道具。
- 发卡/尾巴状态。
- 字幕/台词。
- 音效。

### Step 7：生成与剪辑

优先保证：

- 前 3 秒有强画面或强句子。
- 每 5-8 秒有一次状态变化。
- 字幕短、密度高、能截图。
- 结尾不要拖。

### Step 8：发布前自检

发布前问：

- 这条的钩子够不够？
- 中间有没有掉节奏？
- 用户能不能看懂呼噜噜为什么这样？
- 有没有强化至少一个 IP 记忆点？
- 评论区用户能不能接话？

### Step 9：发布后复盘

记录：

- 完播率。
- 3 秒留存。
- 评论关键词。
- 用户最喜欢的台词。
- 是否有人提到“像我”“打工人”“没电”“周一”等共鸣词。
- 是否有人记住发卡、咖啡、黑眼圈、躺平专员、啊呜呜。

## 4. 四大栏目创作抓手

### 呼噜噜看热点

核心吸引力：热点 + 打工人态度。

流程：

1. 找热点。
2. 判断打工人关心点。
3. 写出呼噜噜替大家说的话。
4. 选择最适合的表达形式。
5. 用 IP 记忆点收尾。

### 呼噜噜的怪早晨

核心吸引力：每天醒来都不一样 + 魔幻视觉。

流程：

1. 选择一种打工人情绪。
2. 把情绪变成可视化形态。
3. 让呼噜噜醒来发现异常。
4. 用一天的打工场景消耗她。
5. 回到现实或留下荒诞结尾。

### 打工喵生存报告

核心吸引力：用户觉得“这就是我”。

流程：

1. 选择一个真实打工人瞬间。
2. 放大呼噜噜的低电量反应。
3. 加入咖啡、发卡、工牌、黑眼圈等记忆点。
4. 结尾落到“还没关机”的坚持。

### 呼噜噜与啊呜呜

核心吸引力：朋友关系 + 情绪分享。

流程：

1. 呼噜噜先独自承受一个情绪。
2. 啊呜呜出现，用碎嘴或投喂打破沉默。
3. 两人互相嫌弃，但最后一起续命。
4. 不要求温情过度，朋友感要自然。

## 5. 完播设计规则

- 3 秒内必须出现热点、反常画面、强共鸣台词之一。
- 5-8 秒必须有第一次变化。
- 12-18 秒必须升级一次。
- 45 秒后开始收尾，不要再开新信息。
- 结尾最好有一句可评论的台词。

## 6. 评论区引导

评论区不是硬问，而是顺着打工人情绪接话。

可用方向：

- “你今天电量还剩几格？”
- “你们工牌上写的是什么？”
- “今天是红色发卡还是灰色发卡？”
- “哪个瞬间让你直接宕机？”
- “啊呜呜这个朋友，你身边有吗？”
