# Review Video Output

## 目的

在发布前检查视频是否符合账号定位，并把失败经验沉淀回项目文件。

## 何时使用

- 视频初版生成后。
- 角色图、表情图、动作图生成后。
- 发布前复核。
- 数据表现差后复盘。

## 输入

```yaml
script: "脚本"
shot_list: "分镜"
asset_refs:
  - "使用的资产"
video_notes: "成片观察或截图说明"
metrics:
  views: "可选"
  completion_rate: "可选"
  comments: "可选"
```

## 输出

- 问题清单。
- 修改建议。
- 是否可发布。
- 可复用资产记录。
- bad case 条目。

## 审核维度

1. 角色一致性：呼噜噜是否低电量，啊呜呜是否碎嘴吃货，双人关系是否成立。
2. 内容节奏：前 3 秒是否抓人，冲突是否单一，结尾是否有记忆点。
3. 画面质量：核心道具是否正确，场景是否清楚，人物是否稳定。
4. 复用价值：是否产生新表情、动作、台词或场景。
5. 商业风险：是否过度负面、攻击具体人/公司、违背账号长期调性。

## 失败处理

- 角色漂移：退回资产或提示词阶段。
- 节奏弱：退回脚本阶段。
- 场景混乱：退回分镜阶段。
- 可复用价值低：标记为一次性内容，不进入资产库。
- 重复出现的问题：写入 `bad_cases/bad_case_log.yaml` 并更新模板。
