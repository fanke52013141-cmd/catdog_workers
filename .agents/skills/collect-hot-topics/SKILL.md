# Collect Hot Topics

## 工作定位

为《呼噜噜看热点》和其他借热点起势的内容建立热点池。重点不是完整报道热点，而是找出打工人关心、能被呼噜噜低电量视角转译、适合 3D 毛绒动画表达的选题候选。

## 输入

```yaml
date: "YYYYMMDD"
sources:
  - "用户提供"
  - "抖音热榜"
  - "微博热搜"
  - "微信热点"
  - "小红书趋势"
  - "B站热梗"
raw_topics:
  - title: ""
    source: ""
    note: ""
```

如果用户没有提供热点，可先基于可访问公开页面、用户给的截图/链接、或让用户补充热榜内容。不要承诺稳定抓取需要登录、授权或动态反爬的平台数据。

推荐数据源：

- 优先参考 `docs/hot-topic-source-research.md`。
- 当前推荐方案是自部署 `imsyy/DailyHotApi`，覆盖微博、抖音、今日头条、知乎、B站、百度等热榜。
- 公开示例接口只能作为临时备选，不作为稳定生产依赖。

## 输出

写入：

```text
outputs/hot_topics/YYYYMMDD_热点池.md
```

内容包括：

- 热点列表。
- 来源。
- 热度描述。
- 打工人相关度。
- 可转译情绪。
- 视觉表现力。
- 初步评分。
- 推荐进入选题 brief 的热点。
- 不建议做的热点及原因。

## 执行步骤

1. 读取 `docs/hot-topic-selection-guide.md`。
2. 如需接口来源，先读取 `docs/hot-topic-source-research.md`。
3. 收集用户给出的热点、DailyHotApi 结果或可访问的公开热点信息。
4. 删除明显不适合的内容：低俗、强冲突、灾难事故、纯八卦且无打工人共鸣。
5. 按 5 个维度初评：热度、打工人相关度、情绪共鸣、IP 转译度、视觉表现力。
6. 用 `templates/hot-topic-pool-template.md` 输出热点池。
7. 推荐 3-10 个最值得进入下一阶段的热点。

## 检查点

- [ ] 每个推荐热点都能说明打工人为什么关心。
- [ ] 每个推荐热点都至少有一个呼噜噜视角。
- [ ] 没有把新闻搬运当成最终内容。
- [ ] 不做自动同步或自动发布。

## 失败处理

- 热点数据不足：标记“待用户提供”或只输出基于现有材料的候选。
- 热点太泛：要求转成具体打工人瞬间。
- 热点敏感：降级为不建议做，或只保留抽象情绪，不指向具体对象。
