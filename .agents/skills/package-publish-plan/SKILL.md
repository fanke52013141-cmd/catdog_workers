# Package Publish Plan

## 工作定位

把一条视频从“制作材料”整理成“可发布前检查的本地发布包”。本 Skill 只输出标题、封面、简介、话题和评论引导，不负责自动同步到抖音或任何外部平台。

## 输入

```yaml
script_file: ""
shot_list_file: ""
prompt_file: ""
column: ""
topic: ""
duration: ""
publish_goal: "热点流量 | 共鸣评论 | IP 记忆 | 商业测试"
```

## 输出

写入：

```text
outputs/publish_packages/YYYYMMDD_栏目_主题.md
```

内容包括：

- 标题候选。
- 封面文案。
- 视频简介。
- 话题标签。
- 评论区引导。
- 发布前检查。

## 执行步骤

1. 读取脚本和选题 brief。
2. 提炼视频的热点信号、共鸣点和 IP 记忆点。
3. 生成 3 个标题候选：热点向、共鸣向、IP 向。
4. 生成 1-2 条封面文案，保证 1 秒可读。
5. 生成简介，不解释过多设定。
6. 生成话题标签，包含栏目、打工人、热点或生活标签。
7. 生成评论区引导，让用户接自己的电量、工牌、周一、下班、朋友关系。
8. 使用 `templates/publish-package-template.md` 输出。

## 检查点

- [ ] 标题不是单纯复述脚本。
- [ ] 封面文案足够短。
- [ ] 评论引导能让用户说自己的经历。
- [ ] 没有承诺自动发布。
- [ ] 没有泄露未确认工具链或素材信息。

## 失败处理

- 标题太普通：加入热点词、反常设定或打工人共鸣。
- 封面太长：压缩到 6-12 个字左右。
- 标签太散：保留 3-6 个核心标签。
