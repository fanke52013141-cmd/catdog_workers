# Select Topic Angle

## 工作定位

把“可以聊的热点/日常想法”转成“值得拍的一条视频”。本 Skill 只做选题角度和创作 brief，不写完整脚本。

## 输入

```yaml
topic: "热点或日常想法"
source: "热点池 | 用户灵感 | 栏目计划"
column: "呼噜噜看热点 | 呼噜噜的怪早晨 | 打工喵生存报告 | 呼噜噜与啊呜呜"
target_duration: "1 分钟以内"
available_assets:
  - ""
constraints:
  - ""
```

## 输出

写入：

```text
outputs/topic_briefs/YYYYMMDD_栏目_主题.md
```

内容包括：

- 一句话表达。
- 选题打分。
- 角色与场景。
- 前 3 秒钩子。
- 中段升级方向。
- 结尾 IP 回收。
- 需要的新资产。
- 风险与边界。

## 执行步骤

1. 读取 `docs/content-production-workflow.md`。
2. 如果是热点，读取 `docs/hot-topic-selection-guide.md`。
3. 判断它属于四大栏目中的哪一个。
4. 写出一句话表达：这条替打工人说什么。
5. 按热度/新鲜度、共鸣强度、IP 适配度、视觉表现力、制作可控度打分。
6. 明确是否需要啊呜呜出场。只有朋友分享、对手戏、投喂、吐槽或情绪托举时才让她出现。
7. 写出钩子、升级、结尾回收。
8. 用 `templates/topic-brief-template.md` 输出。

## 检查点

- [ ] 选题不是一句泛泛主题，而是一条可拍视频。
- [ ] 一句话表达清楚。
- [ ] 前 3 秒有留人点。
- [ ] 结尾能回到 IP 记忆点。
- [ ] 没有提前写成完整脚本。

## 失败处理

- 角度太散：只保留一个核心情绪。
- 热点和 IP 不搭：换成“呼噜噜看到这个热点的反应”，不要强行演完整热点。
- 制作难度太高：先输出轻量版 brief，并把高阶版本标记为备选。
